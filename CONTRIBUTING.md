## How to Contribute

### 1. Fork the repository
Click **Fork** on GitHub to create your own copy.

### 2. Create a new branch
Use a clear, descriptive name:
```bash
git checkout -b feature/your-feature-name
```

### 3. Make your changes
Follow existing patterns and code style guidelines

### 4. Commit with a meaningful message
git commit -m "Add <short-description-of-change>"

### 5. Push your branch
git push origin feature/your-feature-name

### 6. Open a Pull Request (PR)

Explain:
- What the change does
- Why it is needed
- Any screenshots or logs if relevant
- Issue reference (e.g., Closes #13)


### Testing Requirements
- Before submitting a PR, please ensure:
```bash
cd contracts && npx hardhat test
```

### Development Guidelines
- Write clean, readable code with meaningful comments
- Update documentation if behavior or APIs change
- Break features into small atomic commits
- Avoid committing generated files or secrets
